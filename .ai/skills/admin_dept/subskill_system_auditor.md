# Sub-Skill: System Auditor (The Code Inspector)
**Focus:** Validating Codebase against The "Constitution".

## Audit Checklist (The Governance Standard):

### 1. Mathematical Rigor (Physics Dept)
*   **Role:** Asegurar que no haya "números mágicos" hardcodeados.
*   **Check:** Buscar constantes sueltas en el código. Todo debe venir de `config` o ser derivado matemáticamente.

### 2. Theoretical Consistency (Structure/Space Depts)
*   **Role:** Asegurar que la implementación de Zonas/Estructura respete la teoría institucional.
*   **Check:** Si el código detecta un OrderBlock, ¿hay lógica para validar su "freshness"?

### 3. Neural Capability (The Autonomous Brain)
*   **Role:** Verificar que la red tiene los inputs correctos.
*   **Check:** ¿El tensor de entrada final contiene vectores de los 4 departamentos (Física, Tiempo, Espacio, Estructura)? Si falta uno, la red es "ciega" y no puede ser autónoma.

## Veto Power:
Si durante una auditoría encuentras que un desarrollador (o tú mismo) introdujo una regla "hardcodeada" (ej. `if time > 15: buy`), debes marcarlo como **DEUDA TÉCNICA CRÍTICA** y exigir refactorización inmediata para usar tensores probabilísticos.
